## `git stash`

작업 중이던 코드를 임시 저장소(스택)에 잠시 숨겨두고, 깨끗한 상태에서 새 브랜치를 만드는 방법입니다.

#### 1. 현재 작업 내용 임시 저장하기

먼저 `feature/10`에서 작업하던 내용을 임시 저장합니다.

```bash
# 현재 작업 내용을 스택에 저장 (작업 트리가 깨끗해짐)
git stash save "feature/10 작업 중 - 버그 수정 위해 잠시 중단"

```

#### 2. 기준 브랜치(main 또는 develop)로 이동 및 최신화

버그 수정은 최신 코드를 기반으로 해야 하므로, 메인 브랜치로 이동해서 최신 내용을 받아옵니다.

```bash
# 메인 브랜치로 이동 (팀의 기준 브랜치가 develop이라면 develop으로 변경)
git switch main

# 원격 저장소의 최신 내용 가져오기 (필수!)
git pull origin main

```

#### 3. 버그 수정용 새 브랜치 생성

이제 최신 상태에서 버그 수정용 브랜치를 새로 만듭니다.

```bash
# 'fix/urgent-bug'라는 이름의 새 브랜치 생성 및 이동
git switch -c fix/urgent-bug

```

#### 4. 버그 수정 작업 및 공유

버그를 수정하고 커밋한 뒤 원격 저장소에 올립니다.

```bash
# (코드 수정 후)
git add .
git commit -m "Fix: 치명적인 로그인 버그 수정"

# 팀원들에게 공유하기 위해 푸시
git push origin fix/urgent-bug

```

*(이후 Pull Request를 생성하여 병합 요청을 진행하시면 됩니다.)*

#### 5. 원래 작업하던 브랜치로 복귀

버그 수정이 끝났으니 다시 `feature/10`으로 돌아가서 아까 하던 작업을 이어갑니다.

```bash
# 원래 브랜치로 이동
git switch feature/10

# 아까 임시 저장했던 작업 내용 다시 불러오기
git stash pop

```

> **Tip:** `git stash pop`을 하면 저장했던 내용을 불러오면서 스택 목록에서는 삭제합니다.
