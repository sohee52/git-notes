
## `git restore --staged .`

### 역할

* **스테이징 영역(Index)** 을 마지막 커밋 상태로 되돌린다.

### 상태 변화

* `git add` 되어 있던 파일들
  → **staged → modified (변경됨)** 상태로 이동
* **파일 내용은 그대로 유지됨**

### 영향 받는 영역

* ✅ 스테이징 영역
* ❌ 작업 디렉토리 (코드 내용은 변하지 않음)

### 사용 상황

* `git add` 를 잘못했을 때
* 커밋 전에 **스테이징만 전부 취소**하고 싶을 때

### 요약

> **add 취소 (코드는 유지)**

---

## `git restore .`

### 역할

* **작업 디렉토리(Working Tree)** 를 마지막 커밋 상태로 되돌린다.

### 상태 변화

* 수정된 파일 내용이 **모두 삭제됨**
* 마지막 커밋 시점의 코드로 복구됨

### 영향 받는 영역

* ❌ 스테이징 영역
* ✅ 작업 디렉토리

### 사용 상황

* 수정한 코드가 전부 필요 없을 때
* 마지막 커밋 상태로 **완전히 되돌리고 싶을 때**

### 주의 사항

* 커밋되지 않은 변경사항은 **복구 불가**

### 요약

> **코드 수정 내용 전부 삭제**

---

## 두 명령어를 함께 사용할 경우

```bash
git restore --staged .
git restore .
```

### 결과

* 스테이징 영역: 마지막 커밋 상태
* 작업 디렉토리: 마지막 커밋 상태
* 커밋 이후의 모든 변경사항이 **완전히 제거됨**

### 요약

> **마지막 커밋 이후 작업 전부 폐기**
