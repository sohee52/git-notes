- 원격 브랜치 내용으로 로컬 내용 덮어 씌우는 방법
    
    
    | **구분** | **✅ 방법 1 (정석 해결)** | **🚨 방법 2 (강력 / 위험)** |
    | --- | --- | --- |
    | **핵심 목적** | **'Staged'된 변경 사항**까지
    모두 버리기 | **"푸시 안 한 로컬 커밋"**까지
    모두 버리고 덮어쓰기 |
    | **명령어** | `1. git restore --staged .` <br> `2. git restore .` <br> `3. git pull` | `1. git fetch origin`  <br>  `2. git reset --hard origin/브랜치 이름` |
    | **작동 원리** | 1. 'Staged'를 'Unstaged'로 변경 <br> 2. 'Unstaged'를 모두 삭제 <br> 3. 충돌 원인이 사라져 `pull` 성공 | 1. 원격 정보만 다운로드 <br> 2. 로컬 브랜치 상태를 원격 상태로 (로컬 커밋/변경 싹 다 지우고) 강제 리셋 |
    | **결과** | • **안전하게 `pull` 성공** <br> • (푸시 안 한 로컬 커밋은 **보존됨**) | • **로컬 브랜치가 원격과 100% 동일해짐** <br> • (푸시 안 한 로컬 커밋은 **영구 삭제됨**) |
