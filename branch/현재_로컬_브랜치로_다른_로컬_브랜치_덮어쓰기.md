- 로컬 브랜치 덮어쓰는 방법
    - `switch`는 이동할 때 쓰지만, **브랜치 이름을 바꾸거나 삭제하는 등의 관리**는 여전히 **`branch`** 명령어를 사용합니다.
    - `M` (대문자) 옵션을 사용하면 기존에 같은 이름(`week1-2/sohee`)이 있어도 **강제로 덮어쓰면서 이름을 변경**합니다.
    
    ```bash
    # 현재 clean-week1-2 브랜치에 있는지 확인하세요.
    # 명령어: 현재 브랜치 이름을 week1-2/sohee로 변경 (기존 week1-2/sohee는 덮어씌워짐)
    git branch -M week1-2/sohee
    ```
    
    이제 로컬에서는 `week1-2/sohee`가 깔끔한 상태로 변경되었습니다.
    
    원격 저장소(Github)에 반영하기 (`-force`)
    
    로컬의 역사를 새로 썼기 때문에(커밋 해시값들이 전부 바뀌었음), GitHub에 있는 기존 `week1-2/sohee`와 역사가 달라져서 그냥 `push`하면 거부됩니다.
    
    이때는 "내가 지금 로컬에서 만든 역사가 맞으니까 이걸로 강제 업데이트해!"라고 알려줘야 합니다.
    
    ```bash
    # --force (또는 -f) 옵션을 붙여서 강제로 푸시
    git push origin week1-2/sohee --force
    ```
